# 근무표 정리함

## 포함 기능

- 이메일 매직링크 로그인
- 월별 근무표 이미지 저장
- 월별 스케줄 검수 데이터 저장
- 다른 기기에서 로그인 후 월별 근무표 다시 불러오기
- ChatGPT/Gemini 추출 프롬프트 복사
- CSV 파일 업로드 또는 CSV 붙여넣기
- CSV 데이터를 검수표에 자동 반영
- 이미지 OCR 자동 추출 베타
- 검수표 수동 수정
- 일간 / 주간 / 월간 / 오늘 출근 / 휴무일 보기
- 카톡 공유 텍스트 복사
- 엑셀 다운로드
- 근무 코드 설정

## Supabase 설정 필요 항목

1. Database 테이블 생성 SQL 실행
2. `schedule-images` Storage bucket 생성
3. Storage 정책 SQL 실행
4. Authentication > URL Configuration에 Vercel 주소 등록

## 사용 흐름

1. 이메일 입력 후 Google 로그인 보내기
2. 메일에서 Google 로그인 클릭
3. 기준 연도/월 선택
4. 근무표 이미지 업로드
5. 데이터 입력 탭에서 추출 프롬프트 복사
6. ChatGPT/Gemini에 근무표 이미지와 프롬프트를 넣어 CSV 추출
7. 앱의 데이터 입력 탭에 CSV 붙여넣기 또는 업로드
8. CSV를 검수표에 반영 후 확인
9. 저장 버튼 또는 상단 저장 버튼 클릭
10. 월별 보관함에서 저장된 월 불러오기

## 주의

- 이메일 매직링크 로그인을 사용하려면 Supabase Redirect URL에 실제 Vercel 주소를 등록해야 합니다.
- 데이터 입력 탭의 프롬프트는 화면에 길게 표시하지 않고 복사 버튼으로 제공합니다.
- OCR은 베타 기능이므로 권장 방식은 ChatGPT/Gemini로 CSV를 추출한 뒤 앱에 입력하는 방식입니다.
- Supabase Storage bucket 이름은 반드시 `schedule-images`로 만들어야 합니다.


## Google 로그인 적용 안내

이 버전은 기존 정상 작동 버전에서 로그인 방식만 Google OAuth로 변경한 버전입니다.

- GitHub에는 `index.html`, `styles.css`, `app.js`, `README.md` 4개 파일만 업로드하면 됩니다.
- `auth-callback.html`은 사용하지 않습니다.
- Supabase Authentication > Providers > Google에서 Google Provider를 켜고 Client ID / Client Secret을 저장해야 합니다.
- Supabase Authentication > URL Configuration의 Site URL과 Redirect URLs에는 Vercel 앱 기본 주소를 넣습니다.
- Google Cloud OAuth 클라이언트의 승인된 JavaScript 원본에는 Vercel 앱 주소를 넣고, 승인된 리디렉션 URI에는 Supabase Callback URL을 넣습니다.

## v2.4 rollback-safe UI patch
- 로그인 정상 작동 기준 파일에서 UI만 보정했습니다.
- PC 홈 요약 카드를 3열 카드형으로 정리했습니다.
- 주간 보기는 PC에서 7일 카드가 가로로 보이도록 수정했습니다.
- 월간 범례와 카드 색상을 근무=파랑, 휴무=빨강, 연차=노랑으로 분리했습니다.
- 데이터 입력 버튼명을 `검수표에 반영` / `반영 후 저장`으로 명확히 변경했습니다.
- 코드 설정은 PC에서 여러 열 카드 그리드로 표시됩니다.
- OCR 보조의 추출 영역 입력칸이 카드 밖으로 벗어나지 않도록 보정했습니다.

## v3.0 불확실 추정 코드 검수
- 추출 프롬프트에 `AO?`, `BM?`처럼 불확실한 추정값을 표시하는 규칙을 추가했습니다.
- `?`가 붙은 코드는 데이터 확인 탭에서 `불확실` 항목으로 따로 집계됩니다.
- `AO?`처럼 추정값은 유지하고, 원본 확인 후 맞으면 `?`만 지우면 검수 완료됩니다.
- 표 안의 불확실한 셀은 노란색 점선 스타일로 표시됩니다.
- `확인필요`는 판독 불가, `?` 코드는 추정값 있음, 빈칸은 값 없음으로 구분합니다.

## 이번 버전 변경 사항

- 일간 보기의 기본 화면에 근무 타임라인을 추가했습니다.

- 모바일에서는 시간축을 세로형 목록으로 전환해 작은 화면에서도 근무 시작/종료 시간을 비교하기 쉽게 했습니다.
- 일간 화면에서 `타임라인` / `출근시간별` 보기 방식을 전환할 수 있습니다.
- `?`가 붙은 불확실 코드도 코드표에 시간 정보가 있으면 타임라인에 노란색 점선 스타일로 표시됩니다.

## 이번 버전 추가 수정 사항

- 모바일 일간 타임라인을 단순 목록형이 아니라 PC 타임라인을 세로로 돌린 형태로 변경했습니다.
- 모바일에서는 시간축이 왼쪽 세로축으로 표시되고, 직원별 근무 시간이 세로 막대로 표시됩니다.
- 모바일 화면에서도 누가 몇 시부터 몇 시까지 겹쳐 근무하는지 비교할 수 있도록 직원별 막대 영역은 좌우 스크롤됩니다.


## v2.16 모바일 일간 타임라인 폭 정리
- 모바일 세로 타임라인의 시간 눈금을 `06:00` 대신 `6`, `7`처럼 짧게 표시합니다.
- 모바일 근무 막대 안에서는 `07:00~16:00` 시간 문구를 숨기고, 이름과 코드만 작게 표시합니다.
- 모바일 직원 막대 폭과 시간축 폭을 줄여 작은 화면에서 더 많은 직원을 한 번에 비교할 수 있게 조정했습니다.
- PC 타임라인 표시는 기존처럼 유지됩니다.

## v2.17 모바일 일간 타임라인 초압축
- 모바일 세로 타임라인의 왼쪽 시간축 폭을 더 줄였습니다.
- 직원별 근무 막대를 더 얇게 조정했습니다.
- 막대 안의 이름/코드 텍스트는 더 작게 표시하고, 필요하면 막대 밖으로 살짝 튀어나와 보이도록 클리핑을 줄였습니다.
- 한 화면에서 더 많은 근무자를 비교할 수 있도록 모바일 전용 가로 폭을 추가로 압축했습니다.


## v2.17 모바일 타임라인 라벨 정리
- 모바일 일간 타임라인에서 왼쪽 시간축 폭을 더 줄였습니다.
- 근무 막대는 더 얇게 유지하고, 이름/코드는 바 옆에 포스트잇 라벨처럼 붙도록 정리했습니다.
- 라벨은 막대 밖으로 표시되어 모바일에서도 근무 시간 비교 공간을 덜 차지합니다.
